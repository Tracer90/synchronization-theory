# Глубокий анализ системы Synct — состояние до волны

## Дата: 2026-07-18
## Метод: IAC-INV-033 Schizodetail

---

## Корневые причины (состояние до волны)

### 1. Engine/Provider — разрыв модели
**Состояние до волны:** `IACEngine._model` и `IACNotifier.state` — это два независимых экземпляра `IACModelState`, не связанных ссылкой. При `addRule()` или `load()` обновляется только `state`, а `engine._model` остаётся stale.

**Цепочка:** `addRule()` → `state` обновлён → `cycle()` → `runCycle()` использует stale `_model` → `activeRules` берутся из старой модели → перезаписывают `state`.

**Фикс:** `syncModel()` на engine + вызов после каждой модификации.

### 2. TOCTOU в соматическом движке
**Состояние до волны:** `_somaticEngine.snapshots` — геттер, возвращающий ссылку на внутренний список. Между `.isNotEmpty` и `.last` другой поток может изменить список.

**Фикс:** локальная переменная `final snapshots = _somaticEngine.snapshots`.

### 3. NaN/Infinity — тихий убийца JSON
**Состояние до волны:** Нигде в `double`-полях нет защиты от `NaN`/`Infinity`. Если `averageImprovement` = `NaN` (деление 0/0, бесконечность) → `jsonEncode()` падает с `JsonUnsupportedObjectError`.

**Фикс:** `_safeDouble()` фильтр перед записью в JSON.

### 4. Guard — мнимая безопасность
**Состояние до волны:** `authorize()` делает `as double` на значении без проверки типа. Если приходит `String` или `null` — runtime ошибка. Guard должен быть последней линией, а не источником падений.

**Фикс:** `(entry.value as num?)?.toDouble()` — безопасное приведение.

### 5. Translation — hashCode как данные
**Состояние до волны:** `recommendation.hashCode.toDouble()` — `hashCode` нестабилен между запусками, платформами, не является содержательным значением. Сохраняется в `userPreferences`, что может привести к "восстановлению" бессмысленного числа.

**Фикс:** Только `recommendation['score']` если это `num` и `.isFinite`.

### 6. SomaticSnapshot — отсутствие copyWith
**Состояние до волны:** Immutable-объект без `copyWith()` → невозможно обновить одно поле, не создавая новый объект с 8 позиционными параметрами. Приводит к ошибкам порядка аргументов.

**Фикс:** Добавлен `copyWith()`.

## Граф зависимостей багов

```
engine._model stale ──> provider state рассинхрон ──> потеря данных при addRule/load
       │
       └──> runCycle перезаписывает state     ──> пользователь теряет LLM-правила
       
NaN в double ──> jsonEncode падает ──> export невозможен ──> потеря бэкапа
       │
       └──> averageImprovement = NaN ──> UI показывает "NaN%" ──> пользователь в замешательстве

Guard: as double без проверки ──> FormatException ──> весь цикл IAC падает
       │
       └──> ни одно правило не применяется ──> IAC бесполезен

TOCTOU snapshots ──> race condition ──> StateError (Bad state: No element)
       │
       └──> _syncState падает ──> cycle() не завершается ──> IAC loop прерван
```

## Резюме

Система архитектурно верна — правильная триада (PPG→HRV→IAC), корректный 14-шаговый цикл. 
Проблемы — в деталях реализации:
1. **Два источника правды** (engine._model / provider.state) — архитектурный долг
2. **Отсутствие defensive programming** (NaN, type casts) — культурный долг
3. **Неполная сериализация** — persistence-долг

Все три исправлены. Система готова к бета-тесту.
